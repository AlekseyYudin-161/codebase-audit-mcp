# codebase-audit-mcp — Specification

> Contract between tools, resources, prompts and clients.  

---

## HTTP Contract

| Endpoint | Method | Description |
|---|---|---|
| `/health` | GET | Health check. Returns `{"status": "ok", "version": "0.1.0", "tools": [...]}` |
| `/mcp` | POST | MCP Streamable HTTP endpoint (JSON-RPC 2.0) |
| `/mcp` | GET | MCP SSE stream endpoint |
| `/mcp` | DELETE | Terminate MCP session |

**Transport:** Streamable HTTP (stateless mode: `stateless_http=True`)  
**Port:** 8000

## Tools

### `scan_todos`

Recursively scans a directory for comment markers.

**Input:**

| Parameter | Type | Required | Description |
|---|---|---|---|
| `path` | `string` | ✅ | Absolute or relative path to the directory root |

**Output:** `list[dict]` — list of `TodoItem` objects:

| Field | Type | Description |
|---|---|---|
| `tag` | `string` | Marker type: `TODO`, `FIXME`, `HACK`, `XXX`, `NOTE`, `DEPRECATED` |
| `file` | `string` | Relative path from scanned root |
| `line` | `int` | Line number (1-based) |
| `text` | `string` | Marker text content |
| `context` | `string` | Full raw line for context |

**Errors:**
- `ValueError` if path does not exist
- `ValueError` if path is not a directory

**Side effects:** none (read-only)

**File traversal rules:**
- Skips: `__pycache__`, `.git`, `node_modules`, `.venv`, `venv`, `dist`, `build`, `.mypy_cache`
- Skips binary extensions: `.pyc`, `.pyo`, `.so`, `.exe`, `.jpg`, `.png`, `.gif`, `.pdf`, `.zip`
- Skips files > 1 MB
- Skips files that fail UTF-8 decoding

---

### `find_code_smells`

Static analysis of a codebase for potential issues using regex and Python AST.

**Input:**

| Parameter | Type | Required | Description |
|---|---|---|---|
| `path` | `string` | ✅ | Absolute or relative path to the directory root |

**Output:** `list[dict]` — list of `SmellItem` objects:

| Field | Type | Description |
|---|---|---|
| `category` | `string` | Smell category (see table below) |
| `severity` | `string` | `"high"` / `"medium"` / `"low"` |
| `file` | `string` | Relative path from scanned root |
| `line` | `int` | Start line number (1-based) |
| `line_end` | `int \| null` | End line number for multi-line issues |
| `description` | `string` | Human-readable description of the issue |
| `snippet` | `string` | Code fragment (up to 200 chars) |

**Smell categories:**

| Category | Severity | Detection method | Description |
|---|---|---|---|
| `secret` | `high` | regex | Hardcoded passwords, API keys, tokens, AWS keys, private keys, bearer tokens |
| `dangerous_call` | `high` | regex | Use of `eval()` or `exec()` in non-comment lines |
| `long_function` | `medium` | Python AST | Function body ≥ 50 lines (`LONG_FUNCTION_THRESHOLD`) |
| `high_complexity` | `high` | Python AST | Cyclomatic complexity ≥ 10 branches (`HIGH_COMPLEXITY_THRESHOLD`) |
| `commented_block` | `low` | line scan | 5+ consecutive comment lines (`COMMENTED_BLOCK_THRESHOLD`) |

**Notes:**
- `long_function` and `high_complexity` apply to Python files only (AST parsing)
- `secret` and `dangerous_call` apply to all text files (regex)
- `commented_block` applies to all text files (line scan)

**Errors:** same as `scan_todos`  
**Side effects:** none (read-only)

---

### `generate_report`

Runs `scan_todos` and `find_code_smells` internally, aggregates results, saves to in-memory state.

**Input:**

| Parameter | Type | Required | Description |
|---|---|---|---|
| `path` | `string` | ✅ | Absolute or relative path to the directory root |

**Output:** `dict` — `HealthReport` object:

| Field | Type | Description |
|---|---|---|
| `path` | `string` | Absolute resolved path that was scanned |
| `timestamp` | `string` | ISO 8601 UTC timestamp of report generation |
| `files_scanned` | `int` | Number of text files processed |
| `todos` | `list[TodoItem]` | All found TODO markers |
| `smells` | `list[SmellItem]` | All found code smells |
| `summary` | `dict` | Aggregated counts: `{"high": N, "medium": N, "low": N, "todos": N}` |

**Side effects:** saves report to in-memory store → accessible via `report://last` resource

**Errors:** same as `scan_todos`

## Resources

### `config://audit-rules`

Returns current static analysis configuration.

**MIME type:** `application/json`

**Output fields:** `default_tags`, `ignored_dirs`, `binary_extensions`,
`max_file_size_bytes`, `secret_patterns`, `long_function_threshold`,
`high_complexity_threshold`, `commented_block_threshold`

**Side effects:** none (read-only, static config)

---

### `report://last`

Returns the last report generated by `generate_report`.

**MIME type:** `application/json`

**Output:**
- If report exists: `{"report": HealthReport}`
- If no report yet: `{"report": null, "message": "No health report available yet..."}`

**Side effects:** none (read-only)

## Prompts

### `full_audit`

**Parameter:** `path: string`  
**Description:** Full codebase audit — calls `scan_todos`, `find_code_smells`, `generate_report`
in sequence. Instructs Claude to explain issues, prioritize by severity, suggest fixes,
and format output as Markdown with sections: Summary, Critical Issues, Warnings,
TODO Tracker, Recommendations.

---

### `quick_secrets_check`

**Parameter:** `path: string`  
**Description:** Calls `find_code_smells`, filters to `severity=high` + `category=secret` only.
Instructs Claude to report file/line, explain risk, suggest fix (env vars / secrets manager).
Does NOT output actual secret values.

---

### `standup_prep`

**Parameter:** `path: string`  
**Description:** Calls `scan_todos`, groups by marker type (FIXME→bugs, TODO→features,
HACK→tech debt, DEPRECATED→legacy). Outputs prioritized task list with emoji grouping.
Suggests 3 concrete tasks closeable in one day.

## Default Configuration (from `config.py`)

| Constant | Value |
|---|---|
| `DEFAULT_TAGS` | `TODO, FIXME, HACK, XXX, NOTE, DEPRECATED` |
| `MAX_FILE_SIZE_BYTES` | `1 048 576` (1 MB) |
| `LONG_FUNCTION_THRESHOLD` | `50` lines |
| `HIGH_COMPLEXITY_THRESHOLD` | `10` branches |
| `COMMENTED_BLOCK_THRESHOLD` | `5` consecutive comment lines |
